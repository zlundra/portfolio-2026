Phase 2 · Source Baseline Capture
2.1
Capture source row counts and null rates
Run baseline queries against the source database before any migration job runs. Save results to a validation log table. These are your ground truth — every subsequent check compares to these numbers.
baseline_counts.sql · Run against source before migration
T-SQL
-- Capture baseline row counts per entity
INSERT INTO validation_log.dbo.baseline_counts
    (entity_name, row_count, null_pin_count, null_dob_count, captured_at)
SELECT
    'tbl_Registrant',
    COUNT(*),
    SUM(CASE WHEN PINNumber IS NULL THEN 1 ELSE 0 END),
    SUM(CASE WHEN DateOfBirth IS NULL THEN 1 ELSE 0 END),
    GETDATE()
FROM CRM_PROD.dbo.tbl_Registrant
WHERE IsDeleted = 0;
Baseline row counts captured for all entities
Null rates captured for all critical fields
Baseline saved to validation_log with timestamp
Source snapshot date/time recorded in migration log
Owner: BA / Data Engineer
Duration: 2–4 hours
Phase 3 · Field Mapping Verification
3.1
Verify field mapping logic in staging
Before running against production, validate every field mapping in a staging environment. Common issues: data type truncation, encoding differences (UTF-8 vs ASCII), date format mismatches, status value translation tables.
field_mapping_check.sql · Spot check 1,000 records
T-SQL
-- Field mapping spot check: 1,000 random records
SELECT TOP 1000
    src.RegistrantID,
    src.Surname                   AS src_surname,
    tgt.LastName                  AS tgt_surname,
    CASE WHEN src.Surname = tgt.LastName THEN 'MATCH' ELSE 'MISMATCH' END AS surname_check,
    src.DateOfBirth               AS src_dob,
    tgt.Birthdate                 AS tgt_dob,
    CASE WHEN src.DateOfBirth = tgt.Birthdate THEN 'MATCH' ELSE 'MISMATCH' END AS dob_check,
    src.RegistrationStatus        AS src_status,
    tgt.Registration_Status__c    AS tgt_status
FROM CRM_PROD.dbo.tbl_Registrant src
JOIN Staging_SF.dbo.Contact tgt
    ON src.RegistrantID = tgt.Legacy_ID__c
ORDER BY NEWID(); -- random sample
Spot check (1,000 records) run in staging — mismatch rate < 0.5%
Status mapping table verified — all source values have a target mapping
Date format alignment confirmed (ISO 8601 or locale-specific)
Special characters tested (Irish fada: Ó, Á, É, Í, Ú)
ETL mapping config version-controlled and peer-reviewed
Owner: BA + ETL Engineer
Duration: 1 day
Phase 4 · Post-Migration Validation
4.1
Run full automated check suite
Execute all 47 validation checks against the production target. Checks run in order: row counts → referential integrity → field accuracy → null rates → status mapping. Any FAIL halts sign-off.
run_validation_suite.sql · Full check suite
T-SQL
-- Execute full validation suite and write to results table
EXEC validation.dbo.sp_run_full_suite
    @source_db  = 'CRM_PROD_2019',
    @target_db  = 'SF_Production_Snapshot',
    @run_by     = 'valentina@maluxdata.io',
    @run_date   = GETDATE(),
    @threshold_critical    = 0,   -- zero tolerance
    @threshold_noncritical = 5;   -- max 5 row delta

-- Review results
SELECT check_name, entity, status, source_val, target_val, delta
FROM validation.dbo.check_results
WHERE run_date = CAST(GETDATE() AS DATE)
ORDER BY
    CASE status WHEN 'FAIL' THEN 1 WHEN 'WARN' THEN 2 ELSE 3 END;
All row count checks pass (delta = 0 for critical entities)
Referential integrity checks pass (zero orphaned records)
Field accuracy spot check passes (< 0.5% mismatch rate)
Null rate delta within agreed tolerance
All WARN items reviewed and accepted by project sponsor
Exception log produced and shared with stakeholders
Owner: BA Lead
Duration: 2–4 hours

Phase 6 · Post-Go-Live Monitoring
6.1
Daily delta check for first 2 weeks
Run a lightweight row count and referential integrity check daily for the first 14 days post-go-live. Any new delta should be investigated immediately — it may indicate a parallel write to the legacy system or a new ETL issue.
daily_delta_check.sql · Schedule as nightly job
T-SQL
-- Nightly delta check — alert if new discrepancies appear
SELECT
    entity_name,
    row_count                AS current_count,
    baseline_count,
    row_count - baseline_count AS delta_since_golive,
    GETDATE()               AS checked_at
FROM (
    SELECT 'Contact' AS entity_name, COUNT(*) AS row_count
    FROM   SF_Production.dbo.Contact
    WHERE  Legacy_ID__c IS NOT NULL
) curr
JOIN validation_log.dbo.baseline_counts base
    ON curr.entity_name = base.entity_name
WHERE ABS(row_count - baseline_count) > 0;
Daily delta check scheduled as SQL Agent job
Alert configured: email BA lead if delta > 0
Legacy system write access removed (read-only archive)
Post-go-live WARN remediation tasks assigned with deadlines
2-week review meeting scheduled with project team
Migration retrospective completed and documented
Owner: BA + Operations
Duration: 2 weeks monitoring
