Resolution methodology
3-layer framework
Each layer handles progressively more complex matching scenarios. Only records that fail all layers enter the manual queue.
Layer 1
Exact deterministic match
Match on PIN + normalised surname + date of birth. All three must agree exactly. Any mismatch drops to Layer 2. Handles clean records from recent cohorts where data capture was systematic.
31,204 resolved · 72.8% · avg confidence 1.00
Layer 2a
Application status sequence logic
Where PIN is missing or malformed, match on application date range + expected status progression (Applied → Assessed → Registered). Catches pre-2019 records where PINs were not systematically captured.
6,847 resolved · avg confidence 0.91
Layer 2b
Fuzzy name + DOB matching
Levenshtein distance on surname (threshold ≥ 0.88) combined with DOB within a 365-day window. Both conditions must hold simultaneously. Catches hyphenated surname changes, Irish language variants (Ó/O'), and common encoding errors.
4,087 resolved · avg confidence 0.89
Layer 3
Structured manual review queue
Records where Layers 1 and 2 produce conflicting signals or confidence falls below 0.75. Exported to a structured review template with pre-populated candidate matches and conflict reasons. Analyst makes the final determination with documented rationale.
709 flagged · avg 4.2 min per record · ~50 analyst hours total
